# Развёртывание приёмника кликов (webhook.gs)

Это последний недостающий кусок сквозной аналитики. Приёмник ловит момент,
когда человек пришёл из рекламы click-to-Instagram-Direct, и записывает
`ad_id` + IGSID в лист «Клики». Без него блок «люди» на странице «Путь
клиента» остаётся пустым — клики просто негде взять.

Порядок такой: сначала поднять сам приёмник (шаги 1–4), потом подключить
его к Instagram через Meta (шаги 5–8). Раньше времени в Meta не ходите —
там на первом же экране попросят Callback URL, который появится только
после шага 3.

---

## Часть A. Приёмник в Apps Script

### 1. Отдельный проект

Приёмник — это **отдельный проект Apps Script**, не «ФБ». Причина: в «ФБ»
функция `doGet` занята выдачей JSON дашбордам, а Meta для верификации
дёргает `doGet` со своими параметрами — они бы конфликтовали.

- https://script.google.com → **Создать проект**
- Назвать, например, `IG webhook`
- Удалить содержимое `Код.gs`, вставить туда весь [../apps-script/webhook.gs](../apps-script/webhook.gs)
  из репозитория

### 2. Свойства скрипта

Настройки проекта (⚙) → Свойства скрипта → добавить три:

| Свойство | Значение |
|---|---|
| `VERIFY_TOKEN` | любая строка, придумайте (напр. `proznanie-verify-2026`) |
| `URL_SECRET` | любая строка подлиннее (напр. `k9x2mq7p...`) |
| `SHEET_ID` | **тот же** id таблицы, что в проекте «ФБ» — приёмник пишет лист «Клики» в неё же |

`SHEET_ID` берётся из проекта «ФБ» → Настройки → Свойства скрипта → `SHEET_ID`.
Лист «Клики» создастся автоматически при первом событии.

### 3. Развернуть как веб-приложение

- **Начать развертывание** → тип **Веб-приложение**
- «Запуск от имени»: **От моего имени**
- «У кого есть доступ»: **У всех** (Meta ходит без авторизации Google)
- **Развернуть**, при первом разе выдать разрешения проекту

Скопировать **URL веб-приложения** — вида
`https://script.google.com/macros/s/AKfyc.../exec`

### 4. Собрать Callback URL

К URL из шага 3 допишите секрет из шага 2:

```
https://script.google.com/macros/s/AKfyc.../exec?s=<URL_SECRET>
```

Именно этот адрес (с `?s=...`) пойдёт в Meta. Без секрета приёмник
отклоняет запись — это защита от посторонних, знающих голый URL.

> **Почему не подпись Meta.** Apps Script не даёт читать заголовки
> запроса, поэтому проверить `X-Hub-Signature-256` там невозможно.
> Секрет в URL слабее, но рабочая замена. Если однажды понадобится
> настоящая проверка подписи — приёмник переносится на Cloudflare
> Worker, логика остаётся та же.

## Часть B. Подключение в Meta

### 5. Приложение и продукт Instagram

- https://developers.facebook.com → ваше приложение (или создать, тип
  «Бизнес»)
- Добавить продукт **Instagram** → **Настройка вебхуков** (Webhooks)
- Instagram-аккаунт клуба должен быть **профессиональным** и связан с
  страницей Facebook (обычно уже так, раз работает интеграция amoCRM)

### 6. Подписка на webhook

В разделе Webhooks для Instagram:

- **Callback URL**: адрес из шага 4 (с `?s=...`)
- **Verify Token**: значение `VERIFY_TOKEN` из шага 2
- Нажать **Проверить и сохранить** — Meta дёрнет `doGet`, приёмник вернёт
  challenge, галочка позеленеет. Если красный крест — сверьте, что
  `VERIFY_TOKEN` совпадает посимвольно.

### 7. Поле подписки

Подписаться на поле **`messages`** (продукт Instagram). Именно в нём
приходит объект `referral` с `ad_id` для сообщений из рекламы.

### 8. Права доступа

Для чтения переписки нужно разрешение **`instagram_business_manage_messages`**
в режиме **advanced access**. Оно выдаётся через App Review — текст
обоснования и сценарий скринкаста лежат в
[meta-app-review.md](meta-app-review.md).

> **Важно.** До публикации приложения реальные события не приходят
> **никому** — ни администраторам, ни разработчикам, ни тестировщикам.
> Роль тестировщика позволяет настроить подписку и получить маркер, но не
> даёт живых сообщений. Проверить цепочку на собственном клике по рекламе
> до одобрения нельзя. Подробнее — в [meta-app-review.md](meta-app-review.md).

## Проверка, что клики пошли

Возможна **только после публикации приложения** (см. врезку выше). После
одобрения:

1. С другого аккаунта (или с телефона) кликните по активному объявлению
   click-to-Direct и напишите в директ любое сообщение.
2. Откройте таблицу (`SHEET_ID`) → должен появиться лист **«Клики»** со
   строкой: время, IGSID, ad_id, заголовок объявления.
3. Откройте https://jane3dmp.github.io/fb_dashboard/people.html → в блоке
   «люди» появится этот человек, а в таблице объявлений у соответствующего
   объявления счётчик «написали» станет 1.

Если строка в листе есть, а на странице пусто — проверьте, что дата клика
попадает в выбранный период (7/30/90 дней).

Что можно проверить **до** публикации: сам приёмник отвечает на проверочный
запрос Meta. Откройте в браузере, подставив свой `VERIFY_TOKEN` — должно
вернуть `proverka123`:

```
https://script.google.com/macros/s/<ID>/exec?hub.mode=subscribe&hub.verify_token=<VERIFY_TOKEN>&hub.challenge=proverka123
```
